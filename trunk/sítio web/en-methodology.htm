Qooxdoo GUI Builder

The Builder For The New Era Of Web Development

Methodology

In the development of this tool, it was adopted the waterfall model.

 


Analysis

In this phase:

  • Analogous tools were studied and the characteristics to implement where identified;
  • The technologies to be used where defined;
  • The Use Cases diagram was developed;

  • The templates of the same were described;
  • Three possible scenes of use of the application were described.


Drawing

In a general way, in this phase the system was drawn in a way to identify modules of implementation and their characteristics. Specifically:

  • The Classes diagram was elaborated;
  • The State Transition diagram was elaborated;
  • Low-fidelity prototypes of the graphical interfaces of the application, which included the window of the application, as well as dialog windows to confirm actions, were projected;
  • An heuristic evaluation of the low-fidelity prototypes and constant improvements were done.


Implementation

This phase consisted in the codification of the following four modules of implementation:

  • Collect of information about the visual controls to be available to the user to build interfaces and selection of the same;
  • Construction of the graphical interface of the application and implementation of mechanisms to interact with the available visual controls;
  • Construction of a YAML code interpreter and generator that allows the storage and loading of interfaces and templates created by the user;
  • Construction of a HTML code generator that will include a portion of JavaScript code, where, through qooxdoo commands, the interface will be described.


Tests

This phase was useful to verify that the requirements detected during the analysis phase had been implemented.


Maintenance

This phase was not considered during the accomplishment of this project.
